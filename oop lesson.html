شرح دورة أساسيات البرمجة الكائنية (OOP)

البرمجة الكائنية التوجه (OOP) هي أسلوب برمجي يعتمد على الكائنات (Objects)، حيث يتم تمثيل البيانات والوظائف على شكل كائنات تحتوي على خصائص (Attributes) ووظائف (Methods). هذه الدورة ستأخذك في رحلة لتعلم أساسيات البرمجة الكائنية مع أمثلة عملية.

الدرس الأول: مقدمة إلى البرمجة الكائنية (OOP)

البرمجة الكائنية التوجه (OOP) هي نموذج برمجي يعتمد على مفهوم الكائنات (Objects) والفئات (Classes). الكائن هو نسخة من الفئة، ويحتوي على بيانات (خصائص) وسلوكيات (وظائف).

شرح مفصل:
  • الفئة (Class): هي القالب الذي يتم إنشاء الكائنات منه. تحتوي الفئة على خصائص (Attributes) ووظائف (Methods).
  • الكائن (Object): هو نسخة من الفئة، ويحتوي على بيانات وسلوكيات محددة.
  • الخصائص (Attributes): تمثل البيانات التي يحتفظ بها الكائن.
  • الوظائف (Methods): تمثل السلوكيات التي يمكن للكائن تنفيذها.
مثال:
class Car: def __init__(self, brand, color): self.brand = brand self.color = color def drive(self): print(f"The {self.color} {self.brand} is driving.") # إنشاء كائن من الفئة Car my_car = Car("Toyota", "Red") my_car.drive() # The Red Toyota is driving.

الدرس الثاني: التغليف (Encapsulation)

التغليف هو مفهوم يسمح بإخفاء البيانات داخل الكائن وجعل الوصول إليها ممكنًا فقط من خلال وظائف محددة. هذا يساعد على حماية البيانات من التعديل المباشر.

شرح مفصل:
  • الخصائص الخاصة (Private Attributes): لا يمكن الوصول إليها مباشرة من خارج الفئة.
  • الوظائف العامة (Public Methods): تستخدم للوصول إلى البيانات الخاصة أو تعديلها.
مثال:
class Car: def __init__(self, brand, color): self.__brand = brand # خاصية خاصة self.__color = color # خاصية خاصة def get_brand(self): # وظيفة عامة للوصول إلى الخاصية الخاصة return self.__brand my_car = Car("BMW", "Black") print(my_car.get_brand()) # BMW

الدرس الثالث: الوراثة (Inheritance)

الوراثة تسمح بإنشاء فئة جديدة بناءً على فئة موجودة. الفئة الجديدة ترث خصائص ووظائف الفئة الأصلية، ويمكنها إضافة أو تعديل السلوكيات.

شرح مفصل:
  • الفئة الأصلية (Parent Class): الفئة التي يتم الوراثة منها.
  • الفئة الفرعية (Child Class): الفئة التي ترث من الفئة الأصلية.
مثال:
class Vehicle: def __init__(self, brand): self.brand = brand def drive(self): print(f"The {self.brand} is driving.") class Car(Vehicle): # الفئة Car ترث من الفئة Vehicle def __init__(self, brand, color): super().__init__(brand) self.color = color my_car = Car("Toyota", "Red") my_car.drive() # The Toyota is driving.

الدرس الرابع: تعدد الأشكال (Polymorphism)

تعدد الأشكال يسمح للكائنات المختلفة بالاستجابة لنفس الرسالة (Method Call) بطرق مختلفة. هذا يعزز المرونة وإعادة استخدام الكود.

شرح مفصل:
  • تجاوز الوظائف (Method Overriding): إعادة تعريف وظيفة في الفئة الفرعية.
  • تحميل الوظائف (Method Overloading): تعريف أكثر من وظيفة بنفس الاسم ولكن بمعاملات مختلفة.
مثال:
class Animal: def speak(self): pass class Dog(Animal): def speak(self): print("Woof!") class Cat(Animal): def speak(self): print("Meow!") animals = [Dog(), Cat()] for animal in animals: animal.speak() # Woof! Meow!

الدرس الخامس: الواجهات (Interfaces)

الواجهة هي عقد يحدد مجموعة من الوظائف التي يجب أن توفرها الفئة التي تطبقها. هذا يساعد على فصل الوظائف عن التطبيق.

شرح مفصل:
  • تطبيق الواجهة: الفئة التي تطبق الواجهة يجب أن توفر جميع الوظائف المحددة فيها.
مثال:
from abc import ABC, abstractmethod class Shape(ABC): @abstractmethod def area(self): pass class Circle(Shape): def __init__(self, radius): self.radius = radius def area(self): return 3.14 * self.radius ** 2 circle = Circle(5) print(circle.area()) # 78.5

الدرس السادس: الأنماط التصميمية (Design Patterns)

الأنماط التصميمية هي حلول معيارية لمشاكل متكررة في تصميم البرمجيات. تساعد على تحسين جودة الكود وإعادة استخدامه.

شرح مفصل:
  • نمط Singleton: يضمن وجود نسخة واحدة فقط من الفئة.
  • نمط Factory: يوفر واجهة لإنشاء الكائنات دون تحديد الفئة المحددة.
مثال:
class Singleton: _instance = None def __new__(cls): if cls._instance is None: cls._instance = super(Singleton, cls).__new__(cls) return cls._instance obj1 = Singleton() obj2 = Singleton() print(obj1 is obj2) # True

الدرس السابع: إدارة الذاكرة في OOP

إدارة الذاكرة هي عملية تخصيص وإلغاء تخصيص الذاكرة للكائنات. في بعض اللغات مثل Java، يتم إدارة الذاكرة تلقائيًا بواسطة Garbage Collector.

شرح مفصل:
  • Garbage Collector: يتحرر من الذاكرة التي لم تعد مستخدمة.
  • المراجع (References): تحدد كيفية الوصول إلى الكائنات في الذاكرة.
مثال:
class MyClass: def __init__(self, value): self.value = value obj1 = MyClass(10) obj2 = obj1 # obj2 يشير إلى نفس الكائن الذي يشير إليه obj1 print(obj1.value) # 10 print(obj2.value) # 10

الدرس الثامن: الاستثناءات (Exceptions) في OOP

الاستثناءات هي أخطاء تحدث أثناء تنفيذ البرنامج. يمكن التعامل معها باستخدام try-except لمنع توقف البرنامج.

شرح مفصل:
  • try-except: يستخدم للتعامل مع الاستثناءات.
  • finally: يتم تنفيذه دائمًا بغض النظر عن حدوث استثناء أم لا.
مثال:
try: result = 10 / 0 except ZeroDivisionError: print("Cannot divide by zero!") finally: print("This will always execute.")

الدرس التاسع: التطبيقات العملية للبرمجة الكائنية

في هذا الدرس، سنقوم ببناء تطبيق بسيط باستخدام المفاهيم التي تعلمناها في الدورة.

شرح مفصل:
  • تطبيق إدارة الموظفين: سنقوم بإنشاء فئة Employee وإدارة بيانات الموظفين باستخدام OOP.
مثال:
class Employee: def __init__(self, name, salary): self.name = name self.salary = salary def display_info(self): print(f"Name: {self.name}, Salary: {self.salary}") employees = [ Employee("Ali", 5000), Employee("Ahmed", 6000) ] for emp in employees: emp.display_info()
